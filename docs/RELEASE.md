# Release guide

This document describes how to cut a signed production release for Garden Planner.

## Prerequisites

- Android SDK installed and `ANDROID_HOME` set (run `./scripts/doctor.sh` to verify).
- `pnpm install` completed at the repo root.
- A keystore file. If you do not have one yet, generate it once:

```bash
keytool -genkey -v \
  -keystore garden-planner.keystore \
  -alias garden-planner \
  -keyalg RSA -keysize 2048 -validity 10000
```

Store the keystore file outside the repository (e.g. `~/.android/garden-planner.keystore`).

## Step 1 — Set signing env vars

Create `.env.local` at the repo root (it is already git-ignored):

```bash
KEYSTORE_PATH=/absolute/path/to/garden-planner.keystore
KEYSTORE_ALIAS=garden-planner
KEYSTORE_PASSWORD=<your-store-password>
KEY_PASSWORD=<your-key-password>
```

Source the file before running gradle commands:

```bash
set -a && source .env.local && set +a
```

## Step 2 — Bump the version

Open `apps/mobile/app.json` and increment `version` and `android.versionCode`:

```json
{
  "expo": {
    "version": "1.2.0",
    "android": {
      "versionCode": 12
    }
  }
}
```

`versionCode` must be strictly increasing across releases.

## Step 3 — Run the full gate

```bash
pnpm check:all
```

All checks must exit 0 before proceeding.

## Step 4 — Build the production AAB

```bash
pnpm apk:prod
```

This runs `expo prebuild --platform android` followed by `./gradlew bundleRelease`.
The signing block in `android/app/build.gradle` reads the env vars set in Step 1.
The signed AAB is written to:

```
apps/mobile/android/app/build/outputs/bundle/release/app-release.aab
```

## Step 5 — Verify on device

Sideload the AAB or a derived APK onto a real phone and smoke-test the key flows:

- Open the app → home dashboard renders.
- Capture tab → opt-in viewfinder opens, scan completes, verdict fires.
- Sectors tab → sector created from scan.
- Settings → sound toggles, permissions card.

Run the proof sequence:

```bash
scripts/adb-ui.sh alive
scripts/adb-ui.sh shot release-smoke
```

Commit `docs/screenshots/release-smoke.png`.

## Step 6 — Tag the release commit

```bash
git tag v1.2.0
git push origin v1.2.0
```

## Notes

- The `android/` directory is generated by `expo prebuild` and is git-ignored. Never edit it by hand; changes survive only in `app.json`, `app.config.ts`, or Expo plugins.
- EAS Cloud builds (CI) use the `production` profile in `apps/mobile/eas.json`. The local `pnpm apk:prod` script is for offline / air-gapped builds using the same keystore env vars.
- `.env.local` must never be committed. The `.gitignore` at repo root already excludes it.
